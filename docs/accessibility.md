# Purpose

Make accessibility a required quality attribute across every Factor Research surface.

# Target

Meet WCAG 2.2 AA unless a stricter requirement is specified.

# Mandatory requirements

- Semantic HTML and correct heading structure.
- Full keyboard operation.
- Visible and logical focus behavior.
- Accessible names and descriptions.
- Sufficient contrast.
- Reduced-motion support.
- Responsive zoom and text resizing.
- Adequate touch targets.
- Accessible forms, validation, errors, and status announcements.
- Screen-reader-compatible dialogs, menus, tabs, tables, and dynamic updates.

# Financial visualizations

Charts must include textual summaries, accessible data tables where appropriate, keyboard access, and non-color-only indicators.

# Testing

Use automated checks plus manual keyboard, screen-reader, zoom, contrast, and reduced-motion testing.

# AI implementation requirements

The AI must define accessibility behavior before coding a component and include it in tests and acceptance criteria.
